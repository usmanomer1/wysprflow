# Code of Conduct

## Our standard

Participation in this project should stay respectful, direct, and constructive.

Examples of expected behavior:

- focus feedback on the work
- assume good intent, then be specific about problems
- document bugs and tradeoffs clearly
- respect different levels of experience

Examples of unacceptable behavior:

- harassment or personal attacks
- discriminatory language
- doxxing or threats
- deliberate disruption of discussion or review

## Scope

This applies in issues, pull requests, discussions, release conversations, and other project spaces.

## Enforcement

Project maintainers may remove comments, close threads, or block contributors who violate this policy.

## Reporting

Report conduct problems through the private maintainer contact path described in [SECURITY.md](SECURITY.md) when a public issue would make the situation worse.
