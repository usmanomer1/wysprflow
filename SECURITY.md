# Security Policy

## Supported versions

Security fixes are only guaranteed for the latest published release line.

| Version | Supported |
| --- | --- |
| latest | yes |
| older releases | no |

## Reporting a vulnerability

Do not open a public GitHub issue for a security problem.

Preferred path:

1. Use GitHub's private vulnerability reporting or a private security advisory if it is enabled on the repository.
2. If private reporting is not enabled yet, contact the maintainer directly through the contact method listed on the repository profile or project site.

Include:

- affected version
- reproduction steps
- impact
- logs or screenshots if relevant

You should receive an acknowledgement within a few business days. Public disclosure should wait until a fix or mitigation is ready.
